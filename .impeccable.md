# Techyard Systems — Impeccable Design Context

This file persists the design context for Techyard Systems. It's loaded at the start of every design-adjacent session so that visual decisions are consistent without re-establishing the same context from scratch each time.

## Design Context

### Users

Technical buyers doing vendor due diligence on AI agency engagements — operations leads, support leads, CTOs, heads of revenue operations at mid-market companies evaluating whether a custom AI agent is the right next step. They arrive skeptical (the AI agency space is noisy with overclaiming), pressed for time, and looking for evidence of specific outcomes rather than capability claims.

Most will land *deep* on a case study or journal post via search or referral — not on the home page. Every page must stand on its own and convey what the company does and what it's like to work with us.

**The job they're hired for**: decide within a few minutes whether to schedule a 15-minute scoping call — or close the tab.

### Brand Personality

Three words: **confident · proof-first · understated**.

- **Voice**: editorial, not marketing. Example from the existing copy: *"We won't sell you a sixth practice — if the work doesn't fit one of these, a human is probably still the right answer."* No exclamation marks, no "revolutionary", no "synergy", no "10× your results".
- **Emotional goals** within the first five seconds:
  1. **Competence / trust** — this is a real practice, the outcomes are real, I can send them a serious RFP.
  2. **Clarity** — I understand what they do and who they do it for immediately.
  3. **Delight / surprise** — something distinctive (the live orchestration core, a precise turn of phrase) makes me want to read further rather than skim.
- **Anti-personality**: not the SaaS landing page with 50 customer logos and a founder quote. Not the management-consulting firm with stock photography. Not the chatbot demo page with a brightly-colored CTA begging you to "Get started in 60 seconds".

### Aesthetic Direction

**Palette** — single-accent principle. Sage is the ONLY chromatic accent used in chrome.

| Token            | Hex        | Use                                                        |
|------------------|------------|------------------------------------------------------------|
| `paper`          | `#f1ede4`  | Default page background                                    |
| `paper-raised`   | `#fffaf0`  | Cards, form fields, elevated surfaces                      |
| `ink`            | `#2a2f26`  | Primary text, primary buttons                              |
| `ink-muted`      | `#6b7165`  | Secondary text                                             |
| `ink-subtle`     | `#8a7f6b`  | Tertiary / metadata text                                   |
| `accent` (sage)  | `#4a6152`  | Every interactive accent, every italic emphasis            |
| `rule`           | `#e3dccd`  | Hairline borders, rule lines, flax bands                   |
| `ink-deep`       | `#1a1d18`  | Obsidian bands (featured case, stat band, footer)          |

On dark bands, sage reduces to `#a8c0b4` (higher luminance) for legibility. No rust/sky/clay secondary accents in chrome. Icon glyphs for third-party integrations may use brand-adjacent tones internally (Slack's four rectangles, WhatsApp green) — that concession stays inside the SVG.

**Typography**:
- **Display / body serif**: Newsreader, italic reserved for the accent phrase inside a headline (e.g. "Agents that actually *work.*")
- **UI sans**: Space Grotesk — labels, buttons, nav, form inputs.
- **Meta / mono**: JetBrains Mono — eyebrow rails, metadata, code snippets, figure captions.

**Type discipline**:
- Tight negative tracking on headlines (-0.02em to -0.028em)
- Compressed line-heights for display (0.95–1.12); open for body (1.45–1.6)
- Body copy lives at 17px (sans) / 18–19px (serif) with line-height ~1.6

**Layout**:
- Container max 1180px, generous whitespace; section padding via `clamp(80px, 12vw, 160px)`
- Alternating paper/ink bands for cinematic rhythm — a reader should feel they've "turned a page" at each section break
- Corner marks (four L-shaped SVG marks) on dark editorial bands
- No dark mode toggle. Dark bands are in-theme inversions, not a theme variant.

**Signature moments** (earned, not sprinkled):
- **Hero**: `OrchestrationCore` — live hub-and-spoke SVG with 8 integrations radiating from a central "Techyard AI" puck. Proof-by-visualization that we're an integration platform, not a chatbot.
- **Below hero**: `LiveTasksStrip` cycling real-looking outputs every 2.4s.
- **Section 3**: `AgentDemo` — scripted support run with typing bubbles, thought lines, tool calls, and resolution.
- **Case study stat band**: single metric at `clamp(120px, 22vw, 300px)` on ink-deep.
- **Journal posts**: editorial drop cap on first paragraph via `::first-letter`.

### Design Principles

1. **Evidence over adjective.** Show the thing working. If we claim "agents that actually work", the hero runs one. If we claim "68% fewer tickets", that number is the largest glyph on the page. Claims without a measurement don't ship.

2. **Single accent, single moment.** Sage `#4a6152` is the only chromatic accent in chrome. Every page has one hero moment of controlled drama (the live core, the huge metric, the featured case stat). Everything else is quiet by design so that moment lands.

3. **Editorial without affectation.** Keep the serif headlines, the tight tracking, the mono metadata — they carry voice and structure. *Cut* decorative flourishes that don't carry information: masthead volumes (`Vol. IV · No. 02`), "№ 0X" counters on every section eyebrow, fig-captions that say nothing, italic latinate flourishes in microcopy. If it reads like a museum placard, delete it.

4. **Every page stands alone.** Visitors arrive deep via search, journal posts, and referrals. The home page cannot be load-bearing for "what the company does" — every case study, journal post, practice page must convey the brand by itself.

5. **WCAG 2.2 AA is the floor.** All text clears 4.5:1 against its background. Focus rings are visible and consistent (2px solid sage, 2px offset). Motion respects `prefers-reduced-motion`. Interactive elements announce their role correctly (Radix primitives for dialog/accordion/dropdown; `role="checkbox"` for multi-select chips). Alt text is required at schema level for all content images.

### Stack for reference

- **Framework**: Next.js 16.2.4 App Router (Turbopack) · React 19.2.4
- **Styling**: Tailwind CSS 4 with `@theme` tokens mirroring `styles/tokens.css`
- **Content**: Velite + MDX (case studies, journal posts)
- **Primitives**: Radix UI (Dialog for mobile nav, Accordion for security FAQ, Slot for polymorphic buttons)
- **Forms**: React Server Action (`useActionState`) + Zod + Resend
- **Analytics**: Plausible (cookieless, no banner required)
- **Fonts**: Newsreader + Space Grotesk + JetBrains Mono via `next/font`
